Контекст позволяет компонентам получать доступ к значениям, принадлежащим родительским компонентам, без необходимости передавать их через пропсы (избегая передачи через множество промежуточных компонентов, что также называют «пробросом пропсов» (_prop-drilling_)).

Создав пару функций `[get, set]` с помощью `createContext`, вы можете установить контекст в родительском компоненте и получить его в дочернем:

import { Tabs, TabItem } from '@astrojs/starlight/components';

<Tabs syncKey="lang">
  <TabItem label="JS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script>
          import Parent from './Parent.svelte';
          import Child from './Child.svelte';
        </script>

        <Parent>
          <Child />
        </Parent>
        ```
      </TabItem>
      <TabItem label="Parent.svelte">
        ```svelte
        <script>
          import { setUserContext } from './context';

          let { children } = $props();

          setUserContext({ name: 'world' });
        </script>

        {@render children()}
        ```
      </TabItem>
      <TabItem label="Child.svelte">
        ```svelte
        <script>
          import { getUserContext } from './context';

          const user = getUserContext();
        </script>

        <h1>привет, {user.name}, внутри Child.svelte</h1>
        ```
      </TabItem>
      <TabItem label="context.ts">
        ```ts
        import { createContext } from 'svelte';

        interface User {
          name: string;
        }

        export const [getUserContext, setUserContext] = createContext<User>();
        ```
      </TabItem>
    </Tabs>
  </TabItem>
  <TabItem label="TS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script lang="ts">
          import Parent from './Parent.svelte';
          import Child from './Child.svelte';
        </script>

        <Parent>
          <Child />
        </Parent>
        ```
      </TabItem>
      <TabItem label="Parent.svelte">
        ```svelte
        <script lang="ts">
          import { setUserContext } from './context';

          let { children } = $props();

          setUserContext({ name: 'world' });
        </script>

        {@render children()}
        ```
      </TabItem>
      <TabItem label="Child.svelte">
        ```svelte
        <script lang="ts">
          import { getUserContext } from './context';

          const user = getUserContext();
        </script>

        <h1>привет, {user.name}, внутри Child.svelte</h1>
        ```
      </TabItem>
      <TabItem label="context.ts">
        ```ts
        import { createContext } from 'svelte';

        interface User {
          name: string;
        }

        export const [getUserContext, setUserContext] = createContext<User>();
        ```
      </TabItem>
    </Tabs>
  </TabItem>
</Tabs>

:::note
Функция `createContext` была добавлена в версии 5.40. Если вы используете более раннюю версию Svelte, вы должны использовать `setContext` и `getContext` вместо неё.
:::

Это особенно полезно, когда `Parent.svelte` не знает о `Child.svelte` напрямую, а вместо этого отображает его как часть [сниппета](/template-syntax/snippet) `children`, как показано выше.

## `setContext` и `getContext`

В качестве альтернативы `createContext` вы можете использовать `setContext` и `getContext` напрямую. Родительский компонент устанавливает контекст с помощью `setContext(key, value)`...

<Tabs syncKey="lang">
  <TabItem label="JS">
    ```svelte
    // Parent.svelte
    <script>
      import { setContext } from 'svelte';

      setContext('my-context', 'привет из Parent.svelte');
    </script>
    ```
  </TabItem>
  <TabItem label="TS">
    ```svelte
    // Parent.svelte
    <script lang="ts">
      import { setContext } from 'svelte';

      setContext('my-context', 'привет из Parent.svelte');
    </script>
    ```
  </TabItem>
</Tabs>

...а дочерний компонент получает его с помощью `getContext`.

<Tabs syncKey="lang">
  <TabItem label="JS">
    ```svelte
    // Child.svelte
    <script>
      import { getContext } from 'svelte';

      const message = getContext('my-context');
    </script>

    <h1>{message}, внутри Child.svelte</h1>
    ```
  </TabItem>
  <TabItem label="TS">
    ```svelte
    // Child.svelte
    <script lang="ts">
      import { getContext } from 'svelte';

      const message = getContext('my-context');
    </script>

    <h1>{message}, внутри Child.svelte</h1>
    ```
  </TabItem>
</Tabs>

Ключ (`'my-context'` в примере выше) и сам контекст могут быть любыми значениями JavaScript.

:::note
`createContext` является предпочтительным вариантом, так как обеспечивает лучшую типобезопасность и избавляет от необходимости использовать ключи.
:::

Помимо [`setContext`](https://svelte.dev/docs/svelte/svelte#setContext) и [`getContext`](https://svelte.dev/docs/svelte/svelte#getContext), Svelte предоставляет функции [`hasContext`](https://svelte.dev/docs/svelte/svelte#hasContext) и [`getAllContexts`](https://svelte.dev/docs/svelte/svelte#getAllContexts).

## Использование контекста с состоянием

Вы можете хранить реактивное состояние в контексте...

<Tabs syncKey="lang">
  <TabItem label="JS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script>
          import { setCounter } from './context.ts';
          import Child from './Child.svelte';

          let counter = $state({
            count: 0
          });

          setCounter(counter);
        </script>

        <button onclick={() => counter.count += 1}>
          увеличить
        </button>

        <Child />
        <Child />
        <Child />

        <button onclick={() => counter.count = 0}>
          сбросить
        </button>
        ```
      </TabItem>
      <TabItem label="Child.svelte">
        ```svelte
        <script>
          import { getCounter } from './context.ts';

          const counter = getCounter();
        </script>

        <p>{counter.count}</p>
        ```
      </TabItem>
      <TabItem label="context.ts">
        ```ts
        import { createContext } from 'svelte';

        interface Counter {
          count: number;
        }

        export const [getCounter, setCounter] = createContext<Counter>();
        ```
      </TabItem>
    </Tabs>
  </TabItem>
  <TabItem label="TS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script lang="ts">
          import { setCounter } from './context.ts';
          import Child from './Child.svelte';

          let counter = $state({
            count: 0
          });

          setCounter(counter);
        </script>

        <button onclick={() => counter.count += 1}>
          увеличить
        </button>

        <Child />
        <Child />
        <Child />

        <button onclick={() => counter.count = 0}>
          сбросить
        </button>
        ```
      </TabItem>
      <TabItem label="Child.svelte">
        ```svelte
        <script lang="ts">
          import { getCounter } from './context.ts';

          const counter = getCounter();
        </script>

        <p>{counter.count}</p>
        ```
      </TabItem>
      <TabItem label="context.ts">
        ```ts
        import { createContext } from 'svelte';

        interface Counter {
          count: number;
        }

        export const [getCounter, setCounter] = createContext<Counter>();
        ```
      </TabItem>
    </Tabs>
  </TabItem>
</Tabs>

...однако обратите внимание, что если вы _переприсваиваете_ `counter` вместо его обновления, вы «разорвете связь» — другими словами, вместо этого...

```svelte
<button onclick={() => counter = { count: 0 } }>
  сбросить
</button>
```

...вы должны делать так:

```svelte "counter.count = 0"
<button onclick={() => counter.count = 0}>
  сбросить
</button>
```

Svelte предупредит вас, если вы сделаете что-то неправильно.

## Тестирование компонентов

При написании [компонентных тестов](/misc/testing/#юнит-тесты-и-тесты-компонентов-с-vitest) бывает полезно создать обёрточный компонент, который устанавливает контекст, чтобы проверить поведение компонента, который этот контекст использует. Начиная с версии 5.49, можно делать примерно следующее:

```js
import { mount, unmount } from 'svelte';
import { expect, test } from 'vitest';
import { setUserContext } from './context';
import MyComponent from './MyComponent.svelte';

test('MyComponent', () => {
  function Wrapper(...args) {
    setUserContext({ name: 'Вован' });
    return MyComponent(...args);
  }

  const component = mount(Wrapper, {
    target: document.body
  });

  expect(document.body.innerHTML).toBe('<h1>Привет, Вован!</h1>');

  unmount(component);
});
```

Этот подход также работает с [`hydrate`](/runtime/imperative-component-api/#hydrate) и [`render`](/runtime/imperative-component-api/#render).

## Замена глобального состояния

Когда у вас есть состояние, которое используется многими компонентами, может возникнуть соблазн поместить его в отдельный модуль и просто импортировать там, где это необходимо:

```js
// state.svelte.js
export const myGlobalState = $state({
  user: {
    // ...
  }
  // ...
});
```

Во многих случаях это вполне допустимо, но есть риск: если вы изменяете состояние во время рендеринга на стороне сервера (что не рекомендуется, но всё же возможно)...

<Tabs syncKey="lang">
  <TabItem label="JS">
    ```js
    // App.svelte
    <script>
      import { myGlobalState } from 'svelte';

      let { data } = $props();

      if (data.user) {
        myGlobalState.user = data.user;
      }
    </script>
    ```
  </TabItem>
  <TabItem label="TS">
    ```js
    // App.svelte
    <script lang="ts">
      import { myGlobalState } from 'svelte';

      let { data } = $props();

      if (data.user) {
        myGlobalState.user = data.user;
      }
    </script>
    ```
  </TabItem>
</Tabs>

...то данные могут стать доступными для _следующего_ пользователя. Контекст решает эту проблему, так как он не является общим между запросами.
