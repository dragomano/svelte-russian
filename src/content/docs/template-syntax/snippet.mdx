Как и функции, сниппеты могут принимать любое количество параметров, у которых могут быть значения по умолчанию, и вы можете деструктурировать эти параметры. Однако вы не можете использовать остаточные параметры (rest).

## Область видимости сниппетов

Сниппеты могут быть объявлены в любом месте внутри вашего компонента. Они могут ссылаться на значения, объявленные вне их самих, например, в теге `<script>` или в блоках `{#each ...}`...

import { Tabs, TabItem } from '@astrojs/starlight/components';

<Tabs syncKey="lang">
  <TabItem label="JS">
    ```svelte
    // App.svelte
    <script>
      let { message = `Приятно тебя видеть!` } = $props();
    </script>

    {#snippet hello(name)}
      <p>Привет, {name}! {message}!</p>
    {/snippet}

    {@render hello('alice')}
    {@render hello('bob')}
    ```
  </TabItem>
  <TabItem label="TS">
    ```svelte
    // App.svelte
    <script lang="ts">
      let { message = `Приятно тебя видеть!` } = $props();
    </script>

    {#snippet hello(name)}
      <p>Привет, {name}! {message}!</p>
    {/snippet}

    {@render hello('alice')}
    {@render hello('bob')}
    ```
  </TabItem>
</Tabs>

...и они «видимы» для всего в той же области видимости (то есть для соседних элементов и их потомков):

```svelte
<div>
  {#snippet x()}
    {#snippet y()}...{/snippet}

    <!-- это сработает -->
    {@render y()}
  {/snippet}

  <!-- это вызовет ошибку, так как `y` не находится в области видимости -->
  {@render y()}
</div>

<!-- это также вызовет ошибку, так как `x` не находится в области видимости -->
{@render x()}
```

Сниппеты могут ссылаться на себя и друг на друга:

```svelte
// App.svelte
{#snippet blastoff()}
  <span>🚀</span>
{/snippet}

{#snippet countdown(n)}
  {#if n > 0}
    <span>{n}...</span>
    {@render countdown(n - 1)}
  {:else}
    {@render blastoff()}
  {/if}
{/snippet}

{@render countdown(10)}
```

## Передача сниппетов в компоненты

### Явные пропсы

Внутри шаблона сниппеты являются значениями, как и любые другие. Следовательно, их можно передавать в компоненты как пропсы:

### Неявный сниппет `children`

Любой контент внутри тегов компонента, который _не_ является объявлением сниппета, неявно становится частью сниппета `children`:

<Tabs syncKey="lang">
  <TabItem label="JS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script>
          import Button from './Button.svelte';
        </script>

        <Button>нажми меня</Button>
        ```
      </TabItem>
      <TabItem label="Buton.svelte">
        ```svelte
        <script>
          let { children } = $props();
        </script>

        <!-- Результатом будет <button>нажми меня</button> -->
        <button>{@render children()}</button>
        ```
      </TabItem>
    </Tabs>
  </TabItem>
  <TabItem label="TS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script lang="ts">
          import Button from './Button.svelte';
        </script>

        <Button>нажми меня</Button>
        ```
      </TabItem>
      <TabItem label="Buton.svelte">
        ```svelte
        <script lang="ts">
          let { children } = $props();
        </script>

        <!-- Результатом будет <button>нажми меня</button> -->
        ```

      </TabItem>
    </Tabs>

:::note
Обратите внимание, что вы не можете иметь проп с именем `children`, если у вас есть контент внутри компонента. По этой причине следует избегать использования пропсов с таким именем.
:::

### Необязательные пропсы сниппетов

Вы можете объявить пропсы сниппетов как необязательные. Используйте опциональную цепочку, чтобы не отображать ничего, если сниппет не задан...

```svelte
<script>
  let { children } = $props();
</script>

{@render children?.()}
```

...или используйте блок `#if`, чтобы отобразить альтернативный контент:

```svelte
<script>
  let { children } = $props();
</script>

{#if children}
  {@render children()}
{:else}
  альтернативный контент
{/if}
```

## Типизация сниппетов

Сниппеты реализуют интерфейс `Snippet`, импортируемый из `'svelte'`:

```svelte
<script lang="ts">
  import type { Snippet } from 'svelte';

  interface Props {
    data: any[];
    children: Snippet;
    row: Snippet<[any]>;
  }

  let { data, children, row }: Props = $props();
</script>
```

С этим изменением появятся красные волнистые линии, если вы попытаетесь использовать компонент без предоставления пропа `data` и сниппета `row`. Обратите внимание, что аргумент типа, переданный в `Snippet`, является кортежем, так как сниппеты могут иметь несколько параметров.

Мы можем ещё больше улучшить типизацию, объявив универсальный тип, чтобы `data` и `row` ссылались на один и тот же тип:

```svelte
<script lang="ts" generics="T">
  import type { Snippet } from 'svelte';

  let {
    data,
    children,
    row
  }: {
    data: T[];
    children: Snippet;
    row: Snippet<[T]>;
  } = $props();
</script>
```

## Экспорт сниппетов

Сниппеты, объявленные на верхнем уровне файла `.svelte`, могут быть экспортированы из `<script module>` для использования в других компонентах, при условии, что они не ссылаются на объявления в не-модульном `<script>` (непосредственно или косвенно через другие сниппеты):

        ```svelte
        {@render add(1, 2)}
        ```

      <TabItem label="snippets.svelte">
        ```svelte
        <script module>
          export { add };
        </script>

        {#snippet add(a, b)}
          {a} + {b} = {a + b}
        ```

      </TabItem>

## Программные сниппеты

Сниппеты могут быть созданы программно с помощью API [`createRawSnippet`](https://svelte.dev/docs/svelte/svelte#createRawSnippet). Эта функция предназначена для продвинутых случаев использования.

## Сниппеты и слоты

В Svelte 4 содержимое можно передавать компонентам с помощью [слотов](https://svelte.dev/docs/svelte/legacy-slots). Сниппеты более мощные и гибкие, поэтому слоты были объявлены устаревшими в Svelte 5.
