Входные данные компонента называются _пропсами_ (_props_ — сокращение от _properties_). Вы передаете пропсы компонентам так же, как передаете атрибуты элементам:

import { Tabs, TabItem } from '@astrojs/starlight/components';

<Tabs syncKey="lang">
  <TabItem label="JS">
    ```svelte
    // App.svelte
    <script>
      import MyComponent from './MyComponent.svelte';
    </script>

    <MyComponent adjective="cool" />
    ```
  </TabItem>
  <TabItem label="TS">
    ```svelte
    // App.svelte
    <script lang="ts">
      import MyComponent from './MyComponent.svelte';
    </script>

    <MyComponent adjective="cool" />
    ```
  </TabItem>
</Tabs>

С другой стороны, внутри `MyComponent.svelte` мы можем получать пропсы с помощью руны `$props`...

<Tabs syncKey="lang">
  <TabItem label="JS">
    ```svelte
    // MyComponent.svelte
    <script>
      let props = $props();
    </script>

    <p>этот компонент — {props.adjective}</p>
    ```
  </TabItem>
  <TabItem label="TS">
    ```svelte
    // MyComponent.svelte
    <script lang="ts">
      let props = $props();
    </script>

    <p>этот компонент — {props.adjective}</p>
    ```
  </TabItem>
</Tabs>

...хотя чаще всего вы будете [_деструктурировать_](https://developer.mozilla.org/ru/docs/Web/JavaScript/Reference/Operators/Destructuring_assignment) ваши пропсы:

<Tabs syncKey="lang">
  <TabItem label="JS">
    ```svelte "{ adjective }" "{adjective}"
    // MyComponent.svelte
    <script>
      let { adjective } = $props();
    </script>

    <p>этот компонент — {adjective}</p>
    ```
  </TabItem>
  <TabItem label="TS">
    ```svelte "{ adjective }" "{adjective}"
    // MyComponent.svelte
    <script lang="ts">
      let { adjective } = $props();
    </script>

    <p>этот компонент — {adjective}</p>
    ```
  </TabItem>
</Tabs>

## Значения по умолчанию

Деструктуризация позволяет нам объявлять значения по умолчанию, которые используются, если родительский компонент не устанавливает данный проп (или значение `undefined`):

```js
let { adjective = 'happy' } = $props();
```

:::note
Значения по умолчанию не превращаются в прокси реактивного состояния (см. [Обновление пропсов](#обновление-пропсов) для получения дополнительной информации)
:::

## Переименование пропсов

Мы также можем использовать деструктурирующее присваивание для переименования пропсов, что необходимо, если они являются недопустимыми идентификаторами или ключевым словом JavaScript, таким как `super`:

```js
let { super: trouper = 'свет меня найдёт' } = $props();
```

## Остаточные пропсы

Наконец, мы можем использовать _остаточное свойство_ (rest property), чтобы получить все остальные, необъявленные, пропсы:

```js
let { a, b, c, ...others } = $props();
```

## Обновление пропсов

Ссылки на проп внутри компонента обновляются, когда сам проп обновляется — когда `count` изменяется в `App.svelte`, он также изменится внутри `Child.svelte`. Однако дочерний компонент может временно переопределить значение пропа, что может быть полезно для несохранённого эфемерного состояния:

<Tabs syncKey="lang">
  <TabItem label="JS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script>
          import Child from './Child.svelte';

          let count = $state(0);
        </script>

        <button onclick={() => (count += 1)}>
          нажатий (parent): {count}
        </button>

        <Child {count} />
        ```
      </TabItem>
      <TabItem label="Child.svelte">
        ```svelte
        <script>
          let { count } = $props();
        </script>

        <button onclick={() => (count += 1)}>
          нажатий (child): {count}
        </button>
        ```
      </TabItem>
    </Tabs>
  </TabItem>
  <TabItem label="TS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script lang="ts">
          import Child from './Child.svelte';

          let count = $state(0);
        </script>

        <button onclick={() => (count += 1)}>
          нажатий (parent): {count}
        </button>

        <Child {count} />
        ```
      </TabItem>
      <TabItem label="Child.svelte">
        ```svelte
        <script lang="ts">
          let { count } = $props();
        </script>

        <button onclick={() => (count += 1)}>
          нажатий (child): {count}
        </button>
        ```
      </TabItem>
    </Tabs>
  </TabItem>
</Tabs>

Хотя вы можете временно _переопределять_ пропсы, вы не должны их _изменять_, если они не являются [привязываемыми](/runes/bindable/).

Если проп является обычным объектом, изменение не окажет никакого эффекта:

<Tabs syncKey="lang">
  <TabItem label="JS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script>
          import Child from './Child.svelte';
        </script>

        <Child object={{ count: 0 }} />
        ```
      </TabItem>
      <TabItem label="Child.svelte">
        ```svelte
        <script>
          let { object } = $props();
        </script>

        <button onclick={() => {
          // не имеет эффекта
          object.count += 1
        }}>
          нажатий: {object.count}
        </button>
        ```
      </TabItem>
    </Tabs>
  </TabItem>
  <TabItem label="TS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script lang="ts">
          import Child from './Child.svelte';
        </script>

        <Child object={{ count: 0 }} />
        ```
      </TabItem>
      <TabItem label="Child.svelte">
        ```svelte
        <script lang="ts">
          let { object } = $props();
        </script>

        <button onclick={() => {
          // не имеет эффекта
          object.count += 1
        }}>
          нажатий: {object.count}
        </button>
        ```
      </TabItem>
    </Tabs>
  </TabItem>
</Tabs>

Однако, если проп является прокси реактивного состояния, то изменения _будут_ иметь эффект, но вы увидите предупреждение [`ownership_invalid_mutation`](https://svelte.dev/docs/svelte/runtime-warnings#Client-warnings-ownership_invalid_mutation), потому что компонент изменяет состояние, которое ему не «принадлежит»:

<Tabs syncKey="lang">
  <TabItem label="JS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script>
          import Child from './Child.svelte';

          let object = $state({count: 0});
        </script>

        <Child {object} />
        ```
      </TabItem>
      <TabItem label="Child.svelte">
        ```svelte
        <script>
          let { object } = $props();
        </script>

        <button onclick={() => {
          // вызовет обновление счётчика ниже,
          // но с предупреждением. Не изменяйте
          // объекты, которые вам не принадлежат!
          object.count += 1
        }}>
          нажатий: {object.count}
        </button>
        ```
      </TabItem>
    </Tabs>
  </TabItem>
  <TabItem label="TS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script lang="ts">
          import Child from './Child.svelte';

          let object = $state({count: 0});
        </script>

        <Child {object} />
        ```
      </TabItem>
      <TabItem label="Child.svelte">
        ```svelte
        <script lang="ts">
          let { object } = $props();
        </script>

        <button onclick={() => {
          // вызовет обновление счётчика ниже,
          // но с предупреждением. Не изменяйте
          // объекты, которые вам не принадлежат!
          object.count += 1
        }}>
          нажатий: {object.count}
        </button>
        ```
      </TabItem>
    </Tabs>
  </TabItem>
</Tabs>

Стандартное значение пропа, не объявленного с помощью `$bindable`, остается нетронутым — оно не превращается в прокси реактивного состояния — что означает, что изменения не вызовут обновления:

<Tabs syncKey="lang">
  <TabItem label="JS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script>
          import Child from './Child.svelte';
        </script>

        <Child />
        ```
      </TabItem>
      <TabItem label="Child.svelte">
        ```svelte
        <script>
          let { object = { count: 0 } } = $props();
        </script>

        <button onclick={() => {
          // не имеет эффекта, если используется значение по умолчанию
          object.count += 1
        }}>
          нажатий: {object.count}
        </button>
        ```
      </TabItem>
    </Tabs>
  </TabItem>
  <TabItem label="TS">
    <Tabs>
      <TabItem label="App.svelte">
        ```svelte
        <script lang="ts">
          import Child from './Child.svelte';
        </script>

        <Child />
        ```
      </TabItem>
      <TabItem label="Child.svelte">
        ```svelte
        <script lang="ts">
          let { object = { count: 0 } } = $props();
        </script>

        <button onclick={() => {
          // не имеет эффекта, если используется значение по умолчанию
          object.count += 1
        }}>
          нажатий: {object.count}
        </button>
        ```
      </TabItem>
    </Tabs>
  </TabItem>
</Tabs>

В заключение: не изменяйте пропсы. Либо используйте колбэк-пропсы для передачи изменений, либо — если родитель и потомок должны использовать один и тот же объект — используйте руну [`$bindable`](/runes/bindable/).

## Безопасность типов

Вы можете добавить безопасность типов к вашим компонентам, аннотируя ваши пропсы, как вы бы сделали с любым другим объявлением переменной. В TypeScript это может выглядеть так...

```svelte
<script lang="ts">
  let { adjective }: { adjective: string } = $props();
</script>
```

...в то время как в JSDoc вы можете сделать это так:

```svelte
<script>
  /** @type {{ adjective: string }} */
  let { adjective } = $props();
</script>
```

Вы, конечно, можете отделить объявление типа от аннотации:

```svelte
<script lang="ts">
  interface Props {
    adjective: string;
  }

  let { adjective }: Props = $props();
</script>
```

:::note
Интерфейсы для нативных DOM-элементов предоставляются в модуле `svelte/elements` (см. [Типизация обёрток компонентов](/misc/typescript#типизация-обёрток-компонентов)).
:::

Если ваш компонент использует пропсы в виде [сниппетов](/template-syntax/snippet/) (в том числе `children`), [типизируйте их через интерфейс `Snippet`](/template-syntax/snippet/#типизация-сниппетов) из модуля `'svelte'`.

Рекомендуется добавлять типы, так как это подскажет пользователям вашего компонента, какие пропсы они должны предоставить.

## `$props.id()`

Эта руна, появившаяся в версии 5.20.0, создаёт уникальный идентификатор для конкретного экземпляра компонента. При гидратации серверного рендеринга значение сохраняется как на сервере, так и на клиенте.

Это особенно удобно для связывания элементов с помощью атрибутов, таких как `for` и `aria-labelledby`.

```svelte
<script>
  const uid = $props.id();
</script>

<form>
  <label for="{uid}-firstname">First Name: </label>
  <input id="{uid}-firstname" type="text" />

  <label for="{uid}-lastname">Last Name: </label>
  <input id="{uid}-lastname" type="text" />
</form>
```
